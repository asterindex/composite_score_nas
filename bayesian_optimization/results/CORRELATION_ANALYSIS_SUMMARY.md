# 🔬 Кореляційний аналіз Composite Score

## 📊 Проблема старої формули

**Стара формула:**
```
S = -z(L_val) - z(gap) + z(impr) - z(loss_cv) - z(grad_cv) - z(L_tr_last) - z(grad_norm)
```

**Результат:** ρ = -0.23 (слабка негативна кореляція)

---

## 🔍 Кореляція компонентів з Final Val Loss

| Компонент | Spearman ρ | Пояснення |
|-----------|------------|-----------|
| **impr** | **+0.358** | 🚨 Більше improvement → ГІРШИЙ результат! |
| **L_val** | +0.248 | Більше val loss → гірший результат |
| **loss_cv** | +0.216 | Більша варіація → гірший результат |
| **grad_cv** | +0.205 | Більша варіація градієнтів → гірший результат |
| **gap** | +0.183 | Більший gap → гірший результат |
| **L_tr_last** | +0.129 | Більший train loss → гірший результат |
| **grad_norm** | +0.048 | Дуже слабка кореляція |

---

## 💡 Головне відкриття

**Improvement має ПОЗИТИВНУ кореляцію (+0.358)!**

- Моделі що **швидко** покращуються на 2 епохах → **гірший** результат на 30 епохах
- Це пояснює чому прості 2-блочні моделі мали найкращий Composite Score, але найгірший фінальний результат

**Гіпотеза:** Прості моделі швидко навчаються (high improvement), але мають низьку потужність (low capacity) і досягають нижчого максимуму.

---

## 🔧 Нові формули

### Метод 1: Зважені компоненти (НАЙКРАЩИЙ) ✅
```
S = 0.25·impr + 0.20·L_val + 0.15·loss_cv + 0.15·grad_cv + 0.15·gap + 0.05·L_tr_last + 0.05·grad_norm
```

**Результат:** ρ = **+0.301** (покращення на **29.1%**)

**TOP-20 overlap:** 60% (12/20) vs 30% у старій формулі

---

### Метод 2: ТОП-3 компоненти
```
S = 0.5·impr + 0.3·L_val + 0.2·loss_cv
```

**Результат:** ρ = +0.206 (гірше за стару)

---

### Метод 3: Baseline (L_val)
```
S = L_val
```

**Результат:** ρ = +0.248 (трохи краще за стару)

---

## 📝 Для implementation

```python
def calculate_new_composite_score(metrics):
    """Покращена формула Composite Score"""
    score = (
        0.25 * metrics['impr'] +
        0.20 * metrics['L_val'] +
        0.15 * metrics['loss_cv'] +
        0.15 * metrics['grad_cv'] +
        0.15 * metrics['gap'] +
        0.05 * metrics['L_tr_last'] +
        0.05 * metrics['grad_norm_mean']
    )
    return score  # Вище = гірше (мінімізуємо)
```

**Ключова зміна:** ВСІ компоненти з позитивним знаком (бо всі мають позитивну кореляцію з Final Loss)
